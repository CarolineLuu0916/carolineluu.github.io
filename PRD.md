# XINWEI.LU — 个人作品网站 PRD

> **Owner**: 卢欣玮 (Xinwei Lu / Caroline Lu)  
> **定位**: AI 产品经理面试作品集  
> **核心原则**: 视觉优先，内容辅助视觉 —— 网站本身就是设计能力的证明  
> **视觉方向**: 印象派油画风格（柔和光晕、笔触质感、暖调、反 AI 模板审美）

---

## 一、产品概览

### 1.1 网站架构

```
x.inwei.lu (首页 Portal)
├── /trends      → AI Trends（行业情报站）
├── /my          → MY App（车主App元展示）
├── /design      → Design（设计灵感+组件工场）
└── /toy         → Toy（潮玩设计工作台）
```

### 1.2 首页 Portal

首页是 **作品展厅**，不是导航菜单。四个模块以视觉卡片/画框形式呈现，每个卡片：
- 一幅代表性视觉（AI 生成的印象派风格插画或动态背景）
- 模块名称 + 一句话定位
- 悬停时笔触纹理浮现、光晕扩散

不堆文字。让人第一眼记住的是「好看」，第二眼才发现「原来里面内容这么深」。

## 三、模块二：MY App（/my）

> **定位**: 一个「活的产品世界」—— 不展示截图，而是让你走进产品存在的生态  
> **Slogan**: MY — ME, reflected in YOU

### 3.1 三层嵌套结构

```
┌─ 第一层：沙盒世界 ─────────────────────────┐
│  时间流动（现实1分钟 = 世界1小时）             │
│  开发团队动态（PM/开发/设计/测试/运营/运维）    │
│  社区用户：100+ 虚拟用户，自然产生内容          │
│  ┌─ 第二层：虚拟手机 ────────────────────┐   │
│  │  完整手机体验（桌面、通知栏、其他App）    │   │
│  │  ┌─ 第三层：MY App ──────────────┐   │   │
│  │  │  智能助理 / 抽卡 / 社区         │   │   │
│  │  └──────────────────────────────┘   │   │
│  └─────────────────────────────────────┘   │
└─────────────────────────────────────────────┘
```

### 3.2 沙盒世界（第一层）

**时间系统**：加速模式，现实 1 分钟 = 世界 1 小时。30 分钟面试可看到约 1.25 天演进。

**开发团队**（6 个角色）：
- PM（产品经理）— 发布需求、验收功能
- 开发 — 提交代码、修 Bug
- UI/UX 设计 — 发设计稿、改方案
- 测试 — 提 Bug、回归验证
- 运营 — 发活动、看数据
- 运维 — 部署上线、监控

**双重功能**：
- **叙事层**：角色产生工作动态（"抽卡功能上线了""用户反馈UI好酷"），面试官沉浸式感受产品演进
- **文档层**：点击角色可展开真实的 PRD、设计稿、原型等实际产出

**虚拟用户**：100+ 名，各有画像（昵称、头像风格、性格标签、活跃时段）。社区内容按时间轴预生成，面试时看到的是一段时间内自然浮现的内容（5-15 条）。

### 3.3 虚拟手机（第二层）

- 完整手机界面：桌面壁纸 + 应用图标 + 通知栏
- 不同用户的手机有差异化（壁纸、常用 App 排列不同）
- MY App 是其中一个应用图标
- 面试官路径：沙盒用户列表 → 点击用户 → 看到手机桌面 → 点击 MY App 进入

### 3.4 MY App（第三层）

**品牌**：MY = Me & You，原名 AIVA（因面试公司产品同名而改名）

**核心功能**（继承自 AIVA 设计资产，重新包装）：
- **智能助理**：AI 车灵对话，人格化交互
- **抽卡**：今日车运（八卦/玄学抽签）
- **社区**：同灵感应（车主社区）

**视觉**：几何平面可爱 + 纪念碑谷画风。中文界面。

## 四、模块三：Design（/design）

> **定位**: 个人设计灵感库 + 可复用组件工场

### 4.1 灵感收集器

**痛点**：好的设计灵感散落各处，需要一个「个人品味驱动的收藏+探索」系统

**数据源**：
- 自动采集：Dribbble、Mobbin（真实 App 截图）、Awwwards、SiteInspire
- 手动收藏：用户浏览网页/App 时截图保存

**工作流**：
1. 自动爬取 → 粗筛池
2. 用户手动标注喜欢/不喜欢 → 系统逐渐学习视觉偏好
3. 系统根据偏好推荐新的设计灵感

### 4.2 组件工场

**痛点**：截图→代码转化链路断裂，设计难以复用到实际产品

**功能**：
- 从收藏的设计中提取：配色方案、布局模式、动效曲线、字体组合
- 沉淀为可复用代码组件（React 组件 + CSS + 动效参数）
- 标签化管理：按风格/用途/来源检索
- 预览模式：组件在虚拟手机框中实时预览

**愿景**：养成个人设计 Token 库 —— 你开发任何新项目时，可以从这里直接调用已验证的设计组件

## 六、视觉方向

### 6.1 核心审美：印象派 × 穆夏新艺术 × 数字交互

**不是**：赛博朋克、AI 紫蓝渐变、暗黑霓虹  
**而是**：

**底层 — 印象派油画**：
- 莫奈式的柔和光晕、笔触纹理
- 暖调为主，色块交融而非硬边界
- 油画画布质感（CSS 多层渐变 + noise 纹理模拟）
- 自然光感，不刺眼

**上层 — 穆夏 (Alphonse Mucha) 新艺术**：
- 装饰性边框与藤蔓曲线，为内容分区提供结构
- 拜占庭式构图：画框/拱门/圆形徽章作为模块容器
- 金/橄榄/赭石色点缀，呼应版画的金色勾线
- 女性轮廓剪影、植物纹样、几何装饰带
- 分区清晰但边界柔和——不是硬卡片，是「画框中的内容」

**两者关系**：印象派做氛围底，穆夏做结构框。「印象派的柔 + 穆夏的骨」——解决了纯印象派 UI 分区模糊的问题，同时避免了新艺术过于装饰性的累赘感

### 6.2 设计原则

- **视觉优先**：第一眼记住的是画面，不是文字
- **动效丝滑**：页面切换、卡片悬停、内容浮现都要有「笔触晕染」感
- **分区清晰**：四个模块在首页一目了然，但彼此有视觉呼应
- **气质大气**：不像一个「App 列表」，像一个「艺术展厅」
- **反 AI 模板**：不用 Inter/Roboto 字体、不用紫色渐变、不用居中 Hero

### 6.3 字体方向

- 标题：衬线体（呼应油画古典感，如 Playfair Display / Cormorant）
- 正文：人文主义无衬线（如 Newsreader / Source Serif，保持可读性）
- 中文：需要配合印象派气质 —— 偏手写感或宋体/仿宋方向
